  <action>
Create directory `skills/orchestray:learn/` and file `SKILL.md` per D-10, D-11.

The skill definition follows the established skill pattern from `orchestray:specialists`:

```yaml
---
name: learn
description: Extract learning patterns from a completed orchestration
disable-model-invocation: true
argument-hint: [orchestration-id]
---
```

**Skill body — Protocol section:**

1. **Parse arguments**: `$ARGUMENTS`
   - If an orchestration ID is provided (e.g., `orch-1712345678`): use it directly as `{orch-id}`.
   - If empty: find the most recent orchestration by listing directories in `.orchestray/history/`, sorting by name (which contains timestamp), and picking the last one. Report: "Using most recent orchestration: {orch-id}"

2. **Validate:** Check that `.orchestray/history/{orch-id}/events.jsonl` exists using Read tool.
   - If not found: report "No audit trail found for orchestration '{orch-id}'." Then list available orchestrations from `.orchestray/history/` directory. If directory is empty or missing: "No orchestration history found. Run an orchestration first, then use `/orchestray:learn` to extract patterns."

3. **Read the audit trail:** Read `.orchestray/history/{orch-id}/events.jsonl` line by line. Also read `.orchestray/history/{orch-id}/state/task-graph.md` if it exists.

4. **Extract patterns** using the same logic as PM Section 22a automatic extraction:
   - Identify patterns across all four categories: decomposition, routing, specialization, anti-pattern.
   - Decomposition: task breakdown strategies from task-graph.md + orchestration outcome. Success without re-plans = positive pattern. Re-plans = examine what changed for anti-pattern.
   - Routing: routing_outcome events where model choice led to success without escalation = routing pattern. Escalation needed = anti-pattern.
   - Specialization: dynamic_agent_spawn + specialist_saved events where agent succeeded = specialization pattern.
   - Anti-pattern: replan events, verify_fix_fail events, escalation events = what went wrong.
   - Skip extraction for straightforward orchestrations (2-3 tasks, standard flow, no novel insight).
   - Check for duplicates in `.orchestray/patterns/` before creating new patterns.

5. **Show preview** (per D-11) — display extracted patterns in a table BEFORE writing files:
   ```
   | # | Pattern Name | Category | Confidence | Summary |
   |---|-------------|----------|------------|---------|
   | 1 | {name}      | {cat}    | {conf}     | {desc}  |
   ```
   If no patterns identified: "No novel patterns identified from orchestration {orch-id}. The orchestration followed standard patterns already captured or was too straightforward to extract insights from."

6. **Write files:** Create `.orchestray/patterns/{category}-{name}.md` for each pattern using the standard template (YAML frontmatter with name, category, confidence, times_applied, last_applied, created_from, description + markdown body with Context, Approach, Evidence sections). Create `.orchestray/patterns/` directory if it does not exist.

7. **Run pruning** if pattern count > 50: compute `score = confidence * times_applied` for each, remove lowest until count = 50. Report pruned patterns.

8. **Report:** "{N} pattern(s) extracted from orchestration {orch-id}." If patterns were pruned, also report: "Pruned {M} low-value patterns to stay within the 50-pattern cap."
