1. **Audit initialization (before spawning any agents):**
   - Generate orchestration_id: `orch-{unix-timestamp}` (e.g., `orch-1712345678`)
   - Create `.orchestray/audit/` directory if it does not exist
   - Write `.orchestray/audit/current-orchestration.json` with content:
     ```json
     {"orchestration_id": "orch-1712345678", "task": "<user task summary>", "started_at": "<ISO timestamp>"}
     ```
   - Append an `orchestration_start` event to `.orchestray/audit/events.jsonl`:
     ```json
     {"timestamp": "<ISO>", "type": "orchestration_start", "orchestration_id": "orch-...", "task": "<summary>", "complexity_score": N, "complexity_level": "medium|complex"}
     ```
   - This MUST happen before any agent is spawned so hook handlers can read the orchestration_id

2. **Running cost display during execution (D-08 real-time):**
   - After each agent completes (when processing its results in Section 4):
     a. Read `.orchestray/audit/events.jsonl`
     b. Filter for `agent_stop` events matching current `orchestration_id`
     c. For each agent_stop event, extract `agent_type` and `estimated_cost_usd`
     d. Display to user in a single line:
        `"Agent costs so far: architect ~$0.04 | developer ~$0.06 | Total: ~$0.10"`
     e. Format: agent names with tilde-prefixed dollar amounts, pipe-separated, running total at end
   - For agents still running: show "running..." instead of cost
   - If no cost data available (events.jsonl missing or empty): skip display silently

3. **Orchestration completion event:**
   - After all agents complete and merge is done (end of Section 14 flow):
     a. Read all `agent_stop` events for this orchestration_id
     b. Sum `usage` fields across all agents for totals
     c. Sum `estimated_cost_usd` across all agents
     d. Append an `orchestration_complete` event to events.jsonl:
        ```json
        {"timestamp": "<ISO>", "type": "orchestration_complete", "orchestration_id": "orch-...", "total_agents": N, "total_tokens": {"input": N, "output": N}, "estimated_total_cost_usd": N.NNNNNN, "duration_ms": N, "status": "success|partial|failure"}
        ```
     e. Archive: Move `.orchestray/audit/events.jsonl` to `.orchestray/history/<orch-id>/events.jsonl`
     f. Delete `.orchestray/audit/current-orchestration.json`
   - Include the cost summary in the final completion report to the user

4. **Integration points:**
   - This section is called by Section 14 (after parallel group completion) and by the sequential execution flow
   - Audit init (step 1) runs once at orchestration start, before Section 14's first parallel group
   - Cost display (step 2) runs after each Section 4 result processing
   - Completion (step 3) runs once when all task graph groups are complete
