Remove the existing `hooks/.gitkeep` file since hooks.json now exists.

**Part B: Create bin/audit-event.js**

Create directory `bin/` if it does not exist. Create `bin/audit-event.js` as a Node.js script with `#!/usr/bin/env node` shebang. Make it executable (`chmod +x`).

The script must:

1. Read JSON from stdin (buffer all chunks, parse on 'end')
2. Extract from the hook input: `session_id`, `cwd`, `agent_id`, `agent_type`
3. Read the current orchestration_id from `<cwd>/.orchestray/audit/current-orchestration.json` if it exists. If not, use `"unknown"` as orchestration_id. (The PM writes this file before spawning agents — see Plan 03.)
4. Create the audit directory: `<cwd>/.orchestray/audit/` (recursive mkdir)
5. Construct a JSONL event object with these exact fields:
   - `timestamp`: ISO 8601 string (new Date().toISOString())
   - `type`: `"agent_start"`
   - `orchestration_id`: from step 3
   - `agent_id`: from hook input
   - `agent_type`: from hook input
   - `session_id`: from hook input
6. Append the JSON-stringified event + newline to `<cwd>/.orchestray/audit/events.jsonl`
7. Write `{"continue": true}` to stdout
8. Exit 0

Error handling: Wrap all file operations in try/catch. On any error, still output `{"continue": true}` to stdout and exit 0 (never block agent start due to audit failure).

Use ONLY Node.js built-in modules: `fs`, `path`. No external dependencies.

<task type="auto">
  <name>Task 2: Create collect-agent-metrics.js for SubagentStop events with transcript parsing and cost estimation</name>
  <files>bin/collect-agent-metrics.js</files>
  <read_first>bin/audit-event.js (just created — follow same patterns for stdin reading, error handling, JSONL append), .planning/phases/03-automated-execution-and-observability/03-RESEARCH.md (Pattern 4: Transcript-Based Cost Tracking, code example for collect-agent-metrics.js, JSONL event schema)</read_first>
  <action>
Create `bin/collect-agent-metrics.js` as a Node.js script with `#!/usr/bin/env node` shebang. Make it executable (`chmod +x`).

The script must:

1. **Read JSON from stdin** (same pattern as audit-event.js)

2. **Extract from hook input:** `session_id`, `cwd`, `agent_id`, `agent_type`, `agent_transcript_path`, `last_assistant_message`

3. **Read orchestration_id** from `<cwd>/.orchestray/audit/current-orchestration.json` (same as audit-event.js). Default to `"unknown"`.

4. **Parse agent transcript for token usage (D-07 primary source):**
   - If `agent_transcript_path` exists and is readable:
     - Read the file content as UTF-8
     - Split by newlines, filter empty lines
     - For each line, try JSON.parse. If it has a `usage` object, sum the fields:
       - `input_tokens` (accumulate total)
       - `output_tokens` (accumulate total)
       - `cache_read_input_tokens` (accumulate total)
       - `cache_creation_input_tokens` (accumulate total)
     - Skip malformed lines silently (try/catch around each parse)
   - If transcript missing or unreadable: set all usage fields to 0

5. **Estimate cost (D-07 multi-source, Claude's discretion on formula):**
   - Use model-based pricing lookup. Detect model from `agent_type`:
     - If agent_type contains "architect": use sonnet rates (design tasks)
     - If agent_type contains "developer": use sonnet rates (implementation)
     - If agent_type contains "reviewer": use sonnet rates (review)
     - Default: sonnet rates
   - Pricing per 1M tokens (these are estimates — note in comment):
     - sonnet: input $3.00, output $15.00
     - haiku: input $0.25, output $1.25
     - opus: input $15.00, output $75.00
   - Formula:
     ```
     inputCost = (input_tokens / 1_000_000) * rates.input
     outputCost = (output_tokens / 1_000_000) * rates.output
     cacheReadCost = (cache_read_input_tokens / 1_000_000) * rates.input * 0.1
     totalCost = inputCost + outputCost + cacheReadCost
     ```
   - Round to 6 decimal places

6. **Construct JSONL event with these exact fields:**
   - `timestamp`: ISO 8601
   - `type`: `"agent_stop"`
   - `orchestration_id`: from step 3
   - `agent_id`: from hook input
   - `agent_type`: from hook input
   - `session_id`: from hook input
   - `last_message_preview`: first 200 chars of `last_assistant_message` (or empty string)
   - `usage`: object with `input_tokens`, `output_tokens`, `cache_read_input_tokens`, `cache_creation_input_tokens`
   - `estimated_cost_usd`: number from step 5
   - `transcript_path`: `agent_transcript_path` value (or null)

7. **Append** the event to `<cwd>/.orchestray/audit/events.jsonl`

8. **Output** `{"continue": true}` to stdout, exit 0

Error handling: Same as audit-event.js — never block agent stop. If transcript parsing fails entirely, log the event with zero usage and zero cost.

Use ONLY Node.js built-in modules: `fs`, `path`. No external dependencies.
